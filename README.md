 Olist E-Commerce Customer Segmentation

This repository contains the code and methodology for my Capstone Project: **Discovering Hidden Patterns in E-Commerce Using Clustering Techniques**. 

The goal of this project is to perform customer segmentation on the **Olist Brazilian E-Commerce dataset** (over 92,000 unique customers) and extract actionable business insights from a highly imbalanced, mixed-type dataset.

## The Problem
Most customer segmentation pipelines blindly apply **K-Means clustering**. However, K-Means uses Euclidean distance, which assumes all variables are continuous and normally distributed. 

E-commerce data is rarely just continuous. In this dataset, variables like *Purchase Frequency* and *Review Score* are discrete or ordinal. Treating a 1-to-5 star review score as a smooth continuous number creates mathematical artifacts rather than meaningful business segments.

##  The Solution
This project compares a standard K-Means baseline against **K-Prototypes**, an algorithm specifically designed to handle mixed-type data (continuous + categorical) simultaneously. 

### Key Features Used (BAR - Behavioral Analytical Record):
*   **Continuous:** Recency (days), Monetary (BRL), Avg. Delivery Time (days)
*   **Discrete/Ordinal (Categorical):** Frequency (count), Avg. Review Score (1-5), Avg. Installments (count)

##  Key Findings
The K-Prototypes model successfully identified four distinct behavioral personas that were hidden when using basic K-Means:

1.  **High-Value Active Buyers (Cluster 1):** The only group showing repeat purchase behavior, spending 3x more than average. (Retention target)
2.  **Dormant Satisfied Buyers (Cluster 2):** Lapsed customers (>400 days recency) despite having a great delivery and rating experience. (Reactivation target)
3.  **Recent Light-Spenders (Cluster 3):** Very recent, highly satisfied, but low spending. (Upselling target)
4.  **Dissatisfied / Delivery-Affected (Cluster 4):** A critical operational finding. These customers waited **23 days** for delivery (almost 3x longer than others), heavily tanking their review scores. (Logistics fix target)

##  Repository Structure
*   `methodology_analysis.ipynb` / `methodology_analysis.py`: The core data science pipeline (Data merging, Feature Engineering, EDA, K-Means, K-Prototypes, Kruskal-Wallis statistical testing).
*   `figures/`: Contains all 22 visualizations generated by the analysis (Radar charts, PCA scatters, feature distributions, etc.).
*   `*.csv`: The raw Olist datasets used to build the aggregated customer records.

## How to Run
1. Ensure you have the required libraries installed (`pandas`, `scikit-learn`, `kmodes`, `matplotlib`, `seaborn`).
2. Run the `methodology_analysis.py` script or execute the `.ipynb` notebook to preprocess the data, generate the cluster profiles, and output the tracking metrics.
3. The results will be saved as CSVs and visualization PNGs in the `figures/` folder.

The Olist dataset is publicly available on [Kaggle](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce).
